# Pricing and Packaging Brief

## Objective

Define a future-facing packaging model without locking the team into premature monetization decisions.

## This Phase

Research and mock catalog only.

## Key Questions

- what should the free, core, premium, and advanced tiers eventually mean?
- which features are cheap enough to bundle vs expensive enough to meter?
- how should cross-client access be positioned?
- how should mocked premium surfaces map to future commercial plans?

## Required Outputs

- future plan ladder memo
- feature-to-cost mapping
- mock plan catalog structure
- upgrade-path concepts
- “not before” gate tied to billing readiness

## Source Types

- competitor pricing pages
- internal feature cost estimates
- brand positioning decisions
- customer-segment analysis

## Deferred Implementation Gate

Do not implement real paid plans until:

- billing path is selected
- cost estimates are defensible
- support obligations are understood
- corporation and banking setup are complete
